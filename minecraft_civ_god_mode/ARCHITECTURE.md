## Key Design Decisions

### 1. No Duplicates Guarantee
- Uses `Set<string>` for O(1) lookup
- Checks before adding to usedNames
- Fallback to other regions if pool exhausted

### 2. Flexible Distribution
- Configurable percentages per region
- Smart rounding to ensure exact count
- Last region gets remaining bots

### 3. Persistent State
- JSON file stores all mappings
- Can resume from saved state
- Regenerate only when needed

### 4. Team Assignment
- Automatic team detection from role
- Categories: resource, construction, support, artists
- Fallback to 'general' team

### 5. Gender Handling
- 'random' picks from available genders
- 'neutral' uses special pool (unique names)
- Fallback if gender not available in region

## Performance

- **Name Generation**: O(n) where n = bot count
- **Duplicate Check**: O(1) per name (using Set)
- **Query by Name**: O(1) (using Map)
- **Query by Role**: O(1) (using Map)
- **Query by Team**: O(n) (iterates metadata)
